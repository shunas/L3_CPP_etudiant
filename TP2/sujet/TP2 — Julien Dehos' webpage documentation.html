TP2¶

Notions : pointeur, allocation dynamique, fuite mémoire, débogage

Application : liste d’entiers

Mise en place du projet¶

Comme pour le TP précédent, vous allez développer un projet avec un programme principal et un programme de tests unitaires, en utilisant emacs et make.

  • Copiez le projet de base : cp -r TP_vide TP2.
  • Compilez le projet avec cmake et testez les exécutables produits.
  • Quelles sont les étapes à réaliser pour ajouter un nouveau module au projet (fichiers, tests unitaires) ?

Pointeurs et allocation dynamique¶

  • Qu’est-ce-qu’un pointeur ? Quelles sont les quatre fonctionnalités de base des pointeurs ? Dans le programme principal, écrivez du code permettant de tester le scénario suivant :

    • créer un entier a
    • affecter la valeur 42 dans a
    • créer un pointeur d’entier p
    • faire pointer p vers a
    • affecter la valeur 37 dans a via le pointeur

  • Qu’est-ce-que l’allocation dynamique ? Quand l’utiliser ? Dans le programme principal, testez le scénario suivant :

    • créer un pointeur d’entier t
    • allouer un tableau de 10 entiers pointé par t
    • affecter la valeur 42 dans la troisième case du tableau
    • désallouer le tableau
    • réinitialiser le pointeur à nullptr

Implémentation d’une liste chainée¶

On veut implémenter le diagramme de classes suivant.

  • Ajoutez un module Liste à votre projet.
  • Dans ce module, implémentez les structures Noeud et Liste progressivement (en testant chaque méthode avec des tests unitaires, juste après avoir l’avoir écrite). Pour l’instant, ignorez les libérations mémoires.

Analyse des fuites mémoires¶

  • Écrivez un programme principal qui crée une liste contenant les entiers 13 et 37 puis l’affiche en utilisant une boucle for. Vérifiez que votre programme compile et s’exécute.
  • Exécutez votre programme avec valgrind. Que se passe-t-il ? Pourquoi ?
  • Ajoutez un destructeur dans votre module Liste pour résoudre le problème.

Débogage¶

  • Lancez le débogueur. Placez un point d’arrêt dans la méthode getElement et dans le destructeur de Liste puis lancez le débogage.
  • Testez les différentes fonctionnalités du débogueur : pile d’appels, exécution pas-à-pas, inspection de variable…

La régression polynomiale permet d’approximer des données par une fonction polynomiale (droite, parabole…). Soient, par exemple, les données suivantes, issues de mesures physiques imparfaites :

Ces données peuvent être représentées par une régression polynomiale :

  • d’ordre 1 (régression linéaire) : f(x)=−0.247−0.289xf(x)=−0.247−0.289x
  • d’ordre 2 (régression quadratique) : f(x)=−0.997+3.689x−3.485x2f(x)=−0.997+3.689x−3.485x2
  • …

Pour résumer, on a nn données (x,y)(x,y) et on cherche le polynôme ff de degré dd qui approxime le mieux ces données. Écrit sous forme matricielle, le problème revient à résoudre Af=yAf=y pour ff :

⎡⎣⎢⎢⎢⎢⎢11⋮1x0x1⋮xn⋯⋯⋯⋯xd0xd1⋮xdn⎤⎦⎥⎥⎥⎥⎥.⎡⎣⎢⎢⎢⎢f0f1⋮fd⎤⎦⎥⎥⎥⎥=⎡⎣⎢⎢⎢⎢y0y1⋮yn⎤⎦⎥⎥⎥⎥[1x0⋯x0d1x1⋯x1d⋮⋮⋯⋮1xn⋯xnd].[f0f1⋮fd]=[y0y1⋮yn]
